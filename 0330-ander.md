# 论文阅读：QuadGPT——原生四边形网格的自回归生成

**撰稿人：ander**

---

## 0. 基本信息：这篇工作在讨论什么？

- **题目**：QuadGPT: Native Quadrilateral Mesh Generation with Autoregressive Models
- **作者**：Jian Liu, Chunshi Wang, Song Guo, Haohan Weng, Zhen Zhou, Zhiqi Li, Jiaao Yu, Yiling Zhu, Jing Xu, Biwen Lei, Zhuo Chen, Chunchao Guo
- **单位**：Hong Kong University of Science and Technology；Tencent Hunyuan
- **会议**：ICLR 2026
- **年份**：2026
- **链接**：https://arxiv.org/abs/2509.21420

这篇工作讨论的是一个在工业建模里非常实际、但在生成模型里长期被“绕开”的问题：**我们能不能不再先生成三角网格，再靠启发式后处理把它拼成四边形，而是直接生成原生的四边形主导网格？**

作者给出的回答是肯定的。QuadGPT 将 mixed-element mesh（以四边形为主、允许少量三角形）表示成统一的离散 token 序列，并用自回归 Transformer 直接预测该序列。进一步地，作者还提出了一个针对拓扑质量的强化学习微调方法 tDPO，使模型不只是“能生成面”，而是更倾向于生成**连续 edge loop 更清晰、断裂更少、更加接近艺术家建模习惯**的网格。

在我的理解里，这篇论文最值得注意的地方不只是“首次做了 native quad generation”，而是它明确指出：

> **四边形拓扑不应该被当成三角网格生成后的附属修补步骤，而应该被作为生成目标本身直接建模。**

---

## 1. 研究问题：为什么“先生成三角网格再转四边形”不够？

当前许多 3D 生成方法的整体流程，本质上还是：

1. 先通过隐式表示、体素或自回归模型得到几何；
2. 再输出一个三角网格；
3. 最后使用 tri-to-quad 算法，把三角面尽可能合并成四边形。

这个流程的问题在于，**几何形状**和**拓扑结构**在中途被割裂了。

对于专业建模、动画绑定、UV 展开和后续细分曲面处理来说，quad-dominant mesh 的价值不只是“四条边”这么简单，而是它背后的**edge flow**、**局部规则性**与**全局结构可控性**。一旦最开始生成的是无结构三角网格，后面的转换算法即使能提升四边形比例，也往往难以恢复原始应有的拓扑意图，例如：

- edge loop 不连续；
- 局部出现断裂与伪影；
- 几何细节与拓扑质量难以兼顾；
- 在复杂拓扑或 sharp feature 附近不稳定。

因此，这篇论文真正要回答的问题是：

> **四边形网格的“结构性”能否被生成模型直接学出来，而不是事后再补出来？**

---

## 2. 工作目标：作者希望把问题解决到什么程度？

QuadGPT 的目标并不是做一个传统意义上的重网格化算法，而是做一个**条件生成模型**：输入一个带法向的点云，输出一个适合生产流程的 quad-dominant mesh。

更具体地说，作者想实现三件事：

1. **端到端生成**：直接从条件输入生成原生四边形主导网格；
2. **兼容真实数据分布**：因为艺术家制作的网格通常并非纯四边形，而是“以四边形为主、夹杂少量三角形”，因此模型要支持 mixed topology；
3. **兼顾几何与拓扑**：不仅要拟合形状，还要生成更连贯、更符合人类偏好的 edge flow。

所以这项工作的目标并不是“把 quad ratio 刷到最高”，而是更接近：

> **生成既保真、又具有可用拓扑结构的 production-ready mesh。**

---

## 3. 主要挑战：原生四边形生成为什么难？

我认为这篇论文面对的挑战主要有四个。

### 3.1 表示层面的挑战：三角形和四边形怎么统一表示？

自回归模型擅长处理定长 token 序列，但 mesh face 的边数并不统一。三角形是 3 个顶点，四边形是 4 个顶点，如果直接序列化，模型会同时面对变长结构与复杂拓扑，训练会比较混乱。

### 3.2 序列长度的挑战：高分辨率网格实在太长

高质量网格通常有大量面片。若每个面都展开为坐标 token，整体序列会非常长，直接用标准 Transformer 建模，计算和显存压力都很大。

### 3.3 学习稳定性的挑战：四边形比三角形更难预测

论文中一个很关键的观察是：**预测一个 quadrilateral face，本质上比预测 triangle face 更复杂**。作者甚至指出，四边形在拓扑上可以近似理解为“两个相关联三角形的联合预测”，因此训练从一开始就直接学 quad-dominant topology，收敛并不稳定。

### 3.4 目标函数的挑战：交叉熵只能保证“句法正确”，不能保证“拓扑好看”

用 cross-entropy 训练自回归模型，可以让模型学会“下一个 token 应该是什么”，但这是一种局部目标。真正决定一个网格是否适合生产使用的，往往是全局出现的拓扑性质，比如：

- edge loop 是否连续；
- fracture（断裂）是否严重；
- 面流是否自然；
- 是否符合艺术家偏好的全局结构。

这类性质很难仅靠逐 token 的监督自动出现。

## 4. 核心思路与方法：QuadGPT 到底怎么做？

### 4.1 统一序列化：把 mixed mesh 写成一个自回归序列

作者首先将 mesh 生成写成一个序列预测问题。设网格为

$$
M = \{f^1, f^2, \dots, f^{N_f}\} \rightarrow \{S(f^1), S(f^2), \dots, S(f^{N_f})\},
$$

其中，$M$ 表示整个网格，$f^j$ 表示第 $j$ 个面，$N_f$ 表示面数，$S(\cdot)$ 表示序列化函数。也就是说，模型不再直接“操作 mesh 数据结构”，而是预测一串离散 token。

为了让表示具有唯一性，作者先将顶点坐标归一化到 $[-0.95, 0.95]^3$，再进行 **1024-level（10 bit）量化**，然后按字典序排序顶点与面，从而建立 mesh 到 token 序列的规范映射。

更关键的是它的 **统一 12-token block 设计**：

- 对四边形面，直接将 4 个顶点的 3D 坐标展开，得到 $4 \times 3 = 12$ 个 token；
- 对三角形面，先补 3 个 padding token，再接 3 个顶点坐标，共同组成 12 个 token。

这样一来，模型不再需要显式的 face type token，而是可以从 padding 的存在与否中隐式识别该面是 triangle 还是 quad。这个设计非常“工程有效”：它简单、统一、可扩展，而且很适合批量并行训练。

### 4.2 Hourglass Transformer：为超长 mesh 序列减压

在模型结构上，作者没有直接套用普通 Transformer，而是采用了 **Hourglass Transformer**。它会先对长序列做逐步压缩，在低分辨率 bottleneck 处建模全局上下文，再逐步恢复到原始长度进行预测。

直观上看，这样做的好处是：

- 外层保留局部几何细节；
- 内层聚合全局拓扑上下文；
- 相比标准 Transformer，更适合处理长 mesh 序列。

此外，QuadGPT 还引入了两类条件信息：

1. **形状条件**：输入是带法向的点云 $P=\{p_i \in \mathbb{R}^6\}_{i=1}^{N_p}$，再通过预训练的 Michelangelo encoder 编成 shape embedding，并通过 cross-attention 注入到解码器中；
2. **quad-dominance 条件**：用一个参数 $r \in [0,1]$ 控制目标网格中四边形主导程度，其中 $r=0$ 对应纯三角训练分布，$r \to 1$ 对应 quad-dominant 分布。

### 4.3 Curriculum learning：先学三角，再学四边形

这篇论文一个非常聪明的训练技巧，是**课程学习**。

作者并不是从零开始直接训练一个 quad 模型，而是：

1. 先用纯三角网格预训练一个基础模型；
2. 再逐渐把训练分布从 triangle-only 平滑过渡到 quad-dominant；
3. 让模型先掌握“几何语法”，再学习更复杂的 quadrilateral topology。

这个思路很符合直觉。因为如果一开始就要求模型学习复杂的四边形拓扑，它往往连最基本的面片生成规律都没学稳；而通过 warm-up，模型先在更简单的问题上收敛，再过渡到更难的四边形结构，训练会稳定得多。

### 4.4 tDPO：让模型直接偏向更好的 edge flow

仅靠交叉熵训练，模型学到的更多是“合法 mesh token 序列的分布”，而不是“艺术家喜欢的 topology”。为此，作者引入了一个强化学习后训练阶段：**truncated Direct Preference Optimization（tDPO）**。

这里的核心思想不是让人工打分，而是构造一个**自动拓扑奖励标准**。作者主要使用了两个量：

- $L_{avg}$：平均连续 edge loop 长度，越大说明面流越连贯；
- $R_{frac}$：fracture 程度，越小说明断裂越少。

模型会对同一输入生成多个候选，再根据这些拓扑奖励自动形成 preference pair，最后用 DPO 风格目标学习“更偏好”那些 edge loop 更清晰、断裂更少的结果。

之所以是 **truncated** DPO，是因为完整 mesh 序列太长，作者在随机前缀和截断窗口上进行训练，从而把 RL 微调真正做到了高分辨率 mesh 场景里可计算、可落地。

在我看来，这一步很关键，因为它把“拓扑好不好”这件事，从视觉直觉转成了可优化的训练信号。也就是说，QuadGPT 不是只在生成后再筛选好结果，而是在训练阶段就把模型往更好的 quad flow 上推。

## 5. 关键实验结果与分析

### 5.1 定量结果：QuadGPT 在几何与拓扑上都很强

论文使用了四类指标：

- **CD**：Chamfer Distance，衡量生成网格与目标几何之间的平均双向距离，越低越好；
- **HD**：Hausdorff Distance，衡量最坏情况下的几何偏差，越低越好；
- **QR**：Quad Ratio，即四边形面所占比例，越高越好；
- **US**：User Study，由专家主观排序打分，越高越好。

在 **Dense Meshes** 上，QuadGPT 取得了：

- **CD = 0.057**
- **HD = 0.147**
- **QR = 80\%**
- **US = 4.9**

在 **Artist Meshes** 上，QuadGPT 取得了：

- **CD = 0.043**
- **HD = 0.095**
- **QR = 78\%**
- **US = 4.8**

与 MeshAnythingV2、BPT、DeepMesh、FastMesh 这类“先生成三角网格，再做 tri-to-quad 转换”的方法相比，QuadGPT 在用户偏好和整体拓扑质量上有明显优势。

这里有一个很值得注意的细节：**QuadriFlow 在成功案例上的 QR 可以到 100\%**，但它在 artist meshes 上的几何误差很大，且失败时会严重退化；论文还特别说明，它的 user study 已经把失败样例按 0 分计入。因此，单独看 QR 并不能说明方法真的更实用。对于生产流程来说，**稳定性和几何保真同样重要**。

### 5.2 定性结果：native generation 明显优于 conversion pipeline

论文中最有说服力的部分之一，是大量可视化对比。

在作者展示的结果里，三角生成基线加转换步骤，常见问题包括：

- 细节区域拓扑混乱；
- edge flow 不自然；
- 局部出现碎裂；
- 转换后虽然“四边形比例上去了”，但整体结构不像艺术家制作的网格。

相比之下，QuadGPT 直接生成的结果在视觉上更规整，面流更连贯，尤其在角色、道具等对 topology 敏感的模型上更明显。论文对 QuadriFlow 的对比也显示：field-guided 方法在复杂拓扑或 sharp feature 上容易不稳定，而 QuadGPT 在这类场景下通常更鲁棒。

### 5.3 消融实验：作者真正证明了什么？

我觉得这篇论文的消融是比较扎实的，因为它不只是说明“加了模块更好”，而是回答了三个关键问题。

#### （1）tDPO 是否真的有用？

答案是肯定的。表 2 中，完整的 **tDPO-Pro** 相比标准 DPO 和简化版 tDPO，在 CD、HD、QR 和用户评分上都更优，说明：

- 只做普通 DPO 不够；
- 必须考虑 truncation；
- 也必须把拓扑奖励设计好，尤其是 edge loop 与 fracture 这两个维度。

#### （2）为什么要课程学习，而不是直接从头训练？

从表 2 看，**From Scratch** 的结果明显差于先三角预训练、再过渡到四边形的 **Finetune** 策略。这说明四边形 topology 的学习难度确实更高，课程学习不是“锦上添花”，而是稳定训练的重要前提。

#### （3）原生生成真的比“先三角后转换”更有必要吗？

这可能是整篇论文最关键的消融。

作者构造了一个很强的基线 **TriGPT**：

- 与 QuadGPT 使用相同的架构；
- 使用同样的 130 万 mesh 数据；
- 也使用同样的 RL 微调策略；
- 唯一区别只是它先生成三角网格，再做 quad 转换。

结果显示，**TriGPT+RL(Q)** 在几何指标上甚至略好于 QuadGPT：

- TriGPT+RL(Q)：CD 0.051，HD 0.138，QR 72\%，US 0.5
- QuadGPT：CD 0.057，HD 0.147，QR 80\%，US 1.3

这说明一个非常有意思的现象：

> **post-hoc conversion 也许还能勉强保住一点几何，但它很难恢复真正好的拓扑。**

换句话说，如果目标只是“表面尽量贴近原几何”，先三角后转换还能竞争；但如果目标是“生成真正可用、具有良好 edge flow 的 quad mesh”，native generation 的优势就非常明显。

## 6. 笔者理解与讨论：这篇工作真正有价值的地方是什么？

### 6.1 它的核心贡献，不只是“刷了一个新 benchmark”

我觉得 QuadGPT 最重要的意义，不只是它在 2026 年把 quad mesh generation 做成了一个新的 SOTA，而是它改变了问题的默认表述方式。

过去很多 3D 生成方法其实默认了一个前提：

> 先把几何形状生成出来，拓扑结构以后再想办法补。

而 QuadGPT 反过来说：

> **如果四边形拓扑本身就是工业应用真正关心的对象，那么它就应该在生成阶段被直接建模。**

这件事听起来像是“表示方式换了一下”，但其实影响很深。因为它意味着：

- topology 不再只是后处理的约束；
- edge flow 不再只是几何生成后的附带修饰；
- mesh generation 终于开始把“结构”当成第一公民。

这和很多图形学问题的直觉其实是吻合的：**高质量网格不是只有几何误差小就够了，拓扑组织方式本身就是质量的一部分。**

### 6.2 这篇工作目前的局限

当然，这篇论文也并不是没有限制。

第一，**它的输入仍然是点云，而不是直接从文本/图像一步到位生成 quad mesh**。严格来说，它更像是“接在上游 3D 生成器之后的高质量 mesh generator”，而不是一个完整的 text-to-quad 统一系统。

第二，**对 out-of-distribution 点云的泛化仍然有限**。作者在失败案例里明确提到：对于来自 AI 资产的 OOD 点云，特别是带有尖锐结构的建筑类模型，模型仍可能出现 fracture 或几何不完整。

第三，**它依然没有完全摆脱“先有 dense shape，再生成结构化 mesh”的链路**。论文自己也承认，一个更理想的方向是未来直接联合生成 geometry 与 topology，而不再绕经中间的 dense mesh。

第四，论文附录里还专门强调了一个很重要的观察：**三角化是不可逆的**。即便使用更好的 tri-to-quad 转换算法，原本 artist-created quad mesh 的 edge flow 一旦在三角化过程中被破坏，后面通常也无法被完美恢复。这一结论反过来也说明了 QuadGPT 的合理性，但同时也提醒我们：如果上游输入本身质量很差，那么 native quad generation 也不会是“万能修复器”。

### 6.3 为什么它依然值得关注？

如果让我用一句话总结这篇论文，我会写成：

> **QuadGPT 证明了，在 mesh generation 里，四边形拓扑可以被“直接生成”，而不是只能在三角网格之后被“尽力挽回”。**

它的价值，不只是做出一个新的模型，而是把一个长期被默认接受的 pipeline——“triangle first, quad later”——第一次真正推到了台前，认真地问了一句：

> **为什么我们不一开始就生成对的拓扑？**

对图形学和 3D 生成来说，这个问题本身就很重要。
